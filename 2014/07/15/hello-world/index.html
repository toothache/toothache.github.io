The following are some simple tests of code formatting, and \(\LaTeX\) support.

Hexo Setup

  1. Install Git
  2. Install Node.js
  3. Intall Hexo
  4. Initial the site folder
  5. Update the site configuration file “_config.yml”
  6. Choose a theme
    The theme of this blog is based on JackMan with some modifications.
  7. Update the theme configuration file “_config.yml”

More Info: Hexo Documentation.

Blog Deployment

Thanks to the GFW, I have to multi-deploy my blog in both GitHub and GitCafe.

  1. Register in GitHub and GitCafe
  2. Generate SSH keys, and add the SSH keys to GitHub and GitCafe
    • Create SSH keys (For Terminal User)
    • Copy the public keys to the clipboard
    • Paste it into GitHub —> Account Settings —> SSH keys
    • Paste it into GitCafe —> 账户设置 —> SSH公钥管理
    • Have a test
  3. Create a new repository named username.github.io in GitHub, where username is your username (or organization name) on GitHub.
  4. Create a new repository in GitCafe.
  5. Create a gitcafe-pages branch in GitCafe.
  6. Edit _config.yml.
    Remember not to leave a space between <repository url> and [branch].
  7. Go to your blog folder and deploy your blog.

Mathematical Blocks

Maxwell’s Equations

\begin{aligned}
\nabla \times \vec{\mathbf{B}} -\, \frac1c\, \frac{\partial\vec{\mathbf{E}}}{\partial t} &= \frac{4\pi}{c}\vec{\mathbf{j}} \\
\nabla \cdot \vec{\mathbf{E}} & = 4 \pi \rho \\
\nabla \times \vec{\mathbf{E}}\, +\, \frac1c\, \frac{\partial\vec{\mathbf{B}}}{\partial t} & = \vec{\mathbf{0}} \\
\nabla \cdot \vec{\mathbf{B}} & = 0
\end{aligned}

Pascal’s rule

$$ \binom{n}{k} = \binom{n-1}{k} + \binom{n-1}{k-1} $$

The Lorentz Equations

\begin{aligned}
\dot{x} & = \sigma(y-x) \\
\dot{y} & = \rho x - y - xz \\
\dot{z} & = -\beta z + xy
\end{aligned}
